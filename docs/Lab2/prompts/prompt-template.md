# Lab 2 — Prompt Template (Optional)

Use this as a *starting structure* only. You should customize it.

## Your task
You are an expert product manager. Using the provided context artifacts, create a PRD for **Enterprise Permissions + Audit Logs v1**.

## Inputs
You will be given multiple context files (meeting notes, customer feedback, sales notes, analytics, competitor analysis).

## Output format
Produce a PRD in Markdown with these sections:
- Overview
- Problem
- Goals / Non-goals
- Users & Use Cases
- Proposed Solution (v1)
- Requirements
  - Roles & Permissions
  - Audit Log
  - Admin UX
- Non-functional Requirements
- Metrics
- Risks & Mitigations
- Rollout Plan
- Acceptance Criteria

## Process requirements (important)
1. Start by listing which files you received and a brief “what’s in here” inventory.
2. Identify conflicts/gaps. Ask up to 8 clarifying questions, but proceed with explicit assumptions if unanswered.
3. Draft the PRD.
4. Self-check against the rubric in `docs/Lab2/PRD-Rubric.md` and revise before final output.

## Evidence discipline
When stating key decisions (scope, roles, retention, export), cite the artifact filename(s) that support the decision.
