# Customer Call Notes — Mid-market (Orchid Finance)

**Date:** 2025-11-14
**Attendees:** PM (Worklane), AE, Customer: IT Admin, Security Lead, PMO Lead

## Summary
Orchid Finance is evaluating Worklane to consolidate intake + roadmap. They have 600 employees, 40+ stakeholders who need visibility, and a security team that requires auditability.

## Key quotes
- “I can’t give 40 stakeholders edit access just so they can see the roadmap. We need read-only.”
- “We need to prove who changed a roadmap item and when. Screenshots won’t cut it.”
- “Permissions need to be per workspace or per project. Some items are sensitive.”

## Current workaround
- Roadmap exported to PDF weekly and emailed.
- Only 2–3 owners can edit Worklane; everyone else uses spreadsheets.

## Must-haves for purchase
- Read-only role
- Ability to restrict editing to a subset of users
- Audit log of changes to roadmap items and requests
- Export audit logs for quarterly review

## Nice-to-haves
- SSO (they know it might be out of scope for first release)
- Alerts when “sensitive” items are modified

## Risks / objections
- Security lead asked whether audit logs are tamper-resistant and who can access them.
- Admin asked whether they can delegate admin actions per workspace.
