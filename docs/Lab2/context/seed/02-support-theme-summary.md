# Support Theme Summary — Permissions & Auditability

**Period:** 2025-10-01 to 2025-12-31
**Source:** Support tickets tagged `permissions`, `access`, `security`, `audit`

## Top themes
1. **Need read-only stakeholders**
   - Execs want visibility without risk of edits.
   - Common for customers with many “viewers”.

2. **Need to restrict editing**
   - Teams want editors limited to a subset (project leads).
   - Accidental edits are frequent; rollback is manual.

3. **Need audit logs**
   - Customers ask “who changed this field?”
   - Security reviews require exporting logs.

4. **Need per-workspace controls**
   - Some customers treat workspaces as departments.
   - Others want per-project segmentation within a workspace.

## Representative issues (paraphrased)
- Members can edit roadmap items they shouldn’t.
- External stakeholders (contractors) need limited access.
- No record of permission changes.

## Product notes
Support repeatedly asks PM for:
- a permissions matrix they can share,
- and a roadmap timeline for audit logs.
